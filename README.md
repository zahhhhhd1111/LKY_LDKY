# LKY_LDKY 征占用林地可研数据整理工具

> 本说明写给**会用 ArcGIS、但不会写代码**的同事。按下面的步骤照做即可，不需要看懂代码。

---

## 一、这个工程是干嘛的

征占用林地可研（LKY_LDKY）要交的材料里，有两类核心数据：

- **ZZY（林地使用图斑）**：从"项目红线"和"林草湿一张图"相交得到。
- **XMHX（项目红线）**：项目本身的占地范围线。

这些原始数据字段乱、不统一，没法直接交。本工程用几段脚本把它们：

1. 整理成**统一标准字段**的图斑；
2. **按县拆开**导出成单独的 shp 文件；
3. **自动填写**到可研附表 Excel 模板里；
4. **自动生成**报告里需要的几段文字。

**最终成果**：在导出结果目录下，每个县一个文件夹，里面有标准化的 `ZZY.shp`、`XMHX.shp`、填好的可研附表 Excel、以及报告文字。

## 三、每个新项目要做的事

### 第 1 步：在 ArcGIS 里手工准备源数据（这步不能用脚本代替）

**① 准备 ZYY 源数据**（项目红线交林草湿一张图、空间连接保护地）：

1. 把项目红线和"林草湿一张图"做相交（Intersect），得到重叠图斑。
2. 对结果做**空间连接**，挂上保护地属性。
3. 检查 `DILEI`（地类）和 `DLMC`（地类名称），把需要备注的内容写到 `BZ` 字段。
4. 用历史红线做**擦除**（Erase），去掉已批过的部分。
5. 多余的 `BZ_1` 之类无关字段不用手动删，脚本会自动忽略。
6. 把整理好的图层**命名为 `多县ZYY空间连接保护区`**，存进 `输出结果.gdb`。

> ⚠️ **注意**：保护区类型字段需要人工核对。当前代码把所有保护区类型都写成了"湿地湖泊类型"，如有不对请手动改正。

**② 准备项目红线（XMHX）源数据**：

1. 把项目红线按标准结构表整理好。
2. 做擦除历史红线处理。
3. **命名为 `多县合并红线_擦除历史`**，存进 `输出结果.gdb`。
4. 这个图层里需要有 `XIAN`（县代码）或 `县代码` 字段，否则没法按县拆分。

### 第 2 步：改配置（每次换项目都要确认）

打开 `project_config.py`，确认这几个名字和你刚放进 gdb 的图层名一致：

```
ZYY_SOURCE_FC_NAME  = "多县ZYY空间连接保护区"      # 第①步做的 ZYY 图层名
ZYY_TARGET_FC_NAME  = "多县ZYY_标准字段版"         # 脚本自动生成的标准版，名字随意
XMHX_SOURCE_FC_NAME = "多县合并红线_擦除历史"       # 第②步做的红线图层名
```

### 第 3 步：按县检查"县代码与投影带"表

打开 `ZYY字段属性标准设置.MD`，找到 **"县代码与投影带"** 这张表。本项目涉及的每个县，都要有：

- `县代码`（6 位）
- `县`（县名）
- `投影带`（`108` / `111` / `114`，对应 CGCS2000 三度带中央经线）

当前规则：望城区、宁乡市、华容县、湘阴县走 **114E**，其余走 **111E**。新县按实际经度填。

## 四、运行脚本（核心操作）

### 4.1 在 ArcMap 的 Python 窗口里跑 1~4 号脚本

打开 ArcMap → 菜单栏 `Geoprocessing` → `Python`（或点工具栏的 Python 图标），弹出 Python 窗口后，**依次**粘贴下面四行，每行回车等它跑完再粘下一行：

```python
exec(open(r"C:\4code\3lot\1-standardZYYshp.py").read())
exec(open(r"C:\4code\3lot\2-standardZYYshpedit.py").read())
execfile(r"C:\4code\3lot\3-export_by_xian.py")
execfile(r"C:\4code\3lot\4-export_xmhx_by_xian.py")
```

每个脚本分别干什么：

| 顺序 | 脚本 | 作用 |
|---|---|---|
| 1 | `1-standardZYYshp.py` | 按 ZYY 标准字段结构，生成一个新的标准图层 `多县ZYY_标准字段版`，把源数据复制进去。 |
| 2 | `2-standardZYYshpedit.py` | 对上一步的图层做 19 步字段整理：单部件化、重算面积、按标准翻译填充字段、检查字段属性。 |
| 3 | `3-export_by_xian.py` | 复制模板目录，按县导出 `林地图斑\ZZY.shp`。 |
| 4 | `4-export_xmhx_by_xian.py` | 按县导出、合并项目红线 `项目红线\XMHX.shp`。 |

跑完 3、4 号脚本后，到导出结果目录（`OUTPUT_BASE`）下，每个县会出现一个文件夹，里面有 `林地图斑\ZZY.shp` 和 `项目红线\XMHX.shp`。

### 4.2 在普通命令行里跑 5、6 号脚本（用 Python 3）

打开 PowerShell 或 cmd，依次运行：

```powershell
python C:\4code\3lot\5-populate_template.py
python C:\4code\3lot\6-写报告.py
```

| 顺序 | 脚本 | 作用 |
|---|---|---|
| 5 | `5-populate_template.py` | 读每个县的 `ZZY.shp`，按字段标准翻译代码，填进可研附表 Excel 模板（A.1、B.1~B.6、费用表），并把县目录打包。 |
| 6 | `6-写报告.py` | 按县生成报告小任务文字（使用林地申请说明、被使用林地单位明细表），输出到 `6-报告小任务结果.md`。 |

## 五、添加新县时要补的东西

除了一开始准备的 ZYY/红线源数据，新增县还要检查：

1. `ZYY字段属性标准设置.MD` 的 **"县代码与投影带"** 表：是否补了该县的 `县代码`、`县`、`投影带`。
2. 同一个 MD 文件：是否补了该县的**乡镇代码、行政村代码**，以及新出现的字段代码字典。
3. `project_config.py` 的 `ZYY_SOURCE_FC_NAME`、`ZYY_TARGET_FC_NAME` 是否指向本次项目。
4. `XMHX_SOURCE_FC_NAME` 是否换成包含新县的红线图层。
5. 新县要按 108E/111E/114E 导出，直接在 MD 表里写对应投影带即可。
6. 如果新县超出现有 108/111/114 三个带的范围，需要补对应模板目录和 `.prj` 投影文件，并扩展代码。

## 六、文件说明

### 核心脚本（按顺序运行）

- `1-standardZYYshp.py`：按标准字段结构生成 ZYY 要素类。
- `2-standardZYYshpedit.py`：单部件化、面积重算、字段填充和字段检查。
- `3-export_by_xian.py`：按县复制模板并导出 `林地图斑\ZZY.shp`。
- `4-export_xmhx_by_xian.py`：按县导出、合并项目红线 `项目红线\XMHX.shp`。
- `5-populate_template.py`：读取各县 ZZY，填充可研附表模板并压缩县目录。
- `6-写报告.py`：按县生成报告小任务文字，输出 Markdown。

### 配置和标准文件

- `project_config.py`：项目路径、源数据名、模板目录等配置（每次换项目要改）。
- `ZYY字段属性标准设置.MD`：字段值翻译、县名/投影带、乡镇/村代码等标准表（加县时要补）。
- `可研附表模板.xlsx`：5 号脚本填充的 Excel 源模板。
- `修改记录_标准ZYY字段.txt`：字段标准的修改记录。

### 辅助脚本

`read_*`、`check_*`、`analyze.py` 等主要用于排查模板和字段标准，不参与主流程。
